# Enterprise AI & Architecture Job Search Portal

A lightweight, high-performance, single-page web application customized explicitly to match executive-level engineering opportunities with candidate profile details. This application runs entirely client-side and is fully optimized for zero-cost deployment via **GitHub Pages**.

## 🎯 Target Geographic Focus
The application filters and tracks matching high-scale positions restricted exclusively to India's primary technology hubs:
* 📍 **Bengaluru (Bangalore)**
* 📍 **Hyderabad**

## 💻 Tech Stack & Architecture
* **Frontend UI:** Semantic HTML5, Responsive CSS3 Grid/Flexbox Layouts, Native Segoe UI typography.
* **Logic Engine:** Vanilla JavaScript (ES6+) managing real-time criteria filtering, string mapping, and dynamic DOM rendering.
* **Server Infrastructure:** Serverless. Fully compiled at run-time directly inside the candidate's browser layer.

## ✨ Core Features
* **Profile Integration:** Displays high-level core competencies directly pulled from the candidate's portfolio.
* **Dual-Core Filter Matrix:** Real-time instantly responsive dropdown toggles allowing cross-filtering by specific City (Bengaluru/Hyderabad) and Domain Mode (Architect Mode, Manager Mode, or Generative AI/RAG implementations).
* **Automated Applying Pipeline:** Pre-configured instantaneous mailto hooks that format job title subjects dynamically to streamline application handshakes.

## 🚀 How to Host This on GitHub (Step-by-Step)

### 1. Create the Repository
1. Log into your GitHub account.
2. Create a new repository and name it `job-tracker` (or any preference).
3. Set the repository visibility to **Public**.

### 2. Upload the Web File
1. Create a file named exactly `index.html` in the root directory of your new repository.
2. Paste the provided web application source code into this file.
3. Commit the changes directly to your `main` or `master` branch.

### 3. Activate GitHub Pages Deployment
1. Inside your repository, click on the **Settings** tab (gear icon at the top menu).
2. Scroll down the left-hand sidebar to the **Code and automation** section and click on **Pages**.
3. Under the **Build and deployment** settings:
   * **Source:** Select `Deploy from a branch`.
   * **Branch:** Choose `main` (or your default branch) and set the folder selector right next to it to `/ (root)`.
4. Click **Save**.

### 4. Access Your Live Portal
GitHub will build your site automatically within 1 to 2 minutes. Your portal will go live globally at your dedicated URL link:
`https://<your-github-username>.github.io/job-tracker/`
